# Lesson 2: Variables (Java)

This lesson demonstrates basic variable usage in Java: declaration, initialization, primitive types, and naming conventions.

What this example shows
- Declaring variables separately and initializing them later (e.g. `int integerVar; integerVar = 42;`)
- Declaring and initializing in one statement (e.g. `double doubleVar = 3.14159;`)
- Primitive types used: `int`, `double`
- Constants using `final` and naming convention `UPPER_SNAKE_CASE` (e.g. `final int CONSTANT_VAR = 100;`)
- Variable naming convention for local variables: `camelCase` (e.g. `myVariable`)

Key points
- Declaration reserves a name and a type; initialization gives it a value.
- Use `final` for values that should not change.
- Follow naming conventions for readability: camelCase for variables, UPPER_SNAKE_CASE for constants.
- Print values with `System.out.println(...)` to verify state during execution.
