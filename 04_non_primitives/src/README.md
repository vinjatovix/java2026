# Lesson 4: Non-Primitive Types (Java)

This lesson demonstrates common non-primitive (reference) types in Java and how to use them via `04_non_primitives/src/com/vinjadevix/lesson04/Main.java` and `04_non_primitives/src/com/vinjadevix/lesson04/ExampleClass.java`.

What this example shows
- Strings: creation, concatenation, common methods (`length()`, `toUpperCase()`, `substring()`, `replace()`, `indexOf()`).
- String formatting: `String.format(...)` and `System.out.printf(...)`.
- Arrays: fixed-size arrays, iteration, accessing and modifying elements, copying to a new array to add elements.
- ArrayList: dynamic array operations (`add`, `remove`) from `java.util.ArrayList`.
- Custom classes: creating a simple class (`ExampleClass`), constructor, getters/setters, and using instance methods.

Key points
- Strings are immutable; methods return new String objects.
- Arrays have fixed length; to change size, create a new array or use collections.
- Use `ArrayList` for dynamic collections of objects (autoboxing used for primitives like `Integer`).
- Custom classes encapsulate data and behavior; use getters/setters and instance methods to interact with objects.
